========================================================================
KEY DESIGN DECISIONS
========================================================================

1. SINGLE QUERY OPTIMIZATION
   Reviews fetched inline with PRs in one paginated GraphQL query.
   Reduced API calls from ~1500 (one per PR) to ~30 (30 PRs per page).

2. DIMINISHING RETURNS
   All raw scores passed through log₂(1 + x) × 10.
   Prevents gaming: 100 small PRs ≠ 100× one PR.

3. CAPPED INPUTS
   PR size capped at log₂ value of 8 (~255 lines).
   Comments capped at 10. Review depth capped at 5.
   Prevents outlier dominance from mega-PRs.

4. WEIGHT NORMALIZATION
   Sidebar sliders auto-normalize shipped + collab + ops = 100%.
   User can explore parameter sensitivity without breaking totals.

5. CACHE STRATEGY
   @st.cache_data(ttl=1800): data fetched once per 30 min.
   Slider changes recompute scores instantly (no API re-fetch).

6. BOT EXCLUSION
   Regex pattern: [bot]$ | dependabot | github-actions.
   Applied to both PR authors and reviewers.
